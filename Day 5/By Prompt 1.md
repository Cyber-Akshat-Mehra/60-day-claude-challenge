# 30-Day Web Development Roadmap (HTML, CSS, JavaScript)

**Goal:** Go from zero to building and deploying a real, responsive, interactive website — no prior coding experience required.

**Daily time commitment:** 1–2 hours
**Final outcome:** A deployed personal portfolio website (multi-page, responsive, interactive) that you can show to employers or clients, plus 4 smaller projects along the way.

---

## Week 1: HTML Foundations — Structure of the Web

**Milestone:** Understand how web pages are structured and build your first static pages.

| Day | Task | Resource |
|---|---|---|
| 1 | Install VS Code + a browser (Chrome/Firefox). Learn what HTML/CSS/JS each do. Write your first `index.html` with headings, paragraphs, and a title. | [MDN: Getting started with HTML](https://developer.mozilla.org/en-US/docs/Learn/HTML/Introduction_to_HTML/Getting_started) |
| 2 | Learn text elements: headings, paragraphs, lists (ordered/unordered), links, line breaks. | [freeCodeCamp: Basic HTML](https://www.freecodecamp.org/learn/2022/responsive-web-design/) |
| 3 | Learn images, attributes, and semantic tags (`header`, `nav`, `main`, `section`, `footer`). | [MDN: HTML semantics](https://developer.mozilla.org/en-US/docs/Glossary/Semantics) |
| 4 | Learn tables and forms (inputs, labels, buttons, select). | [MDN: HTML forms](https://developer.mozilla.org/en-US/docs/Learn/Forms) |
| 5 | Learn `div`, `span`, and how HTML documents nest (the DOM tree concept). | [MDN: HTML basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/HTML_basics) |
| 6 | **Project Day:** Build a one-page personal bio site using only HTML (no styling yet) — include a heading, photo, paragraph about you, a list of hobbies, and a contact form. | — |
| 7 | Review week. Validate your HTML with the W3C validator. Fix errors. Rest/reflect. | [W3C Validator](https://validator.w3.org/) |

**Week 1 Project:** Plain HTML bio page (no CSS).

## Final Outcome

By Day 30, you will have:
- A solid working knowledge of HTML, CSS, and JavaScript fundamentals
- Four real projects in your portfolio, including one live, deployed website
- Comfort with browser DevTools, Git/GitHub, and free deployment tools
- A clear next step (e.g., learning React) to continue growing as a developer

## Tips for Success

- **Code daily, even 20 minutes counts** — consistency beats long, sporadic sessions.
- **Type code by hand** instead of copy-pasting — muscle memory matters early on.
- **Break things on purpose** and fix them — this is the fastest way to really learn.
- **Don't skip the projects** — they're where the real learning happens, not the tutorials.
- **Use DevTools constantly** — inspecting real websites teaches you as much as any course.
